01문서 사용 가이드

1.1 페이지 카드 구성

각 화면은 한 장의 페이지 카드로 정의된다. 카드는 다음 7개 블록을 동일한 순서·동일한 키로 가진다.

블록내용
① 헤더화면 ID(Pxx), 화면명, 라우트(#/...)
② 메타목적 · 진입 경로 · 인증 요구
③ 레이아웃위→아래 섹션 순서, 각 섹션의 라벨과 한 줄 설명
④ 핵심 컴포넌트이 화면에서만 등장하거나 핵심 비중을 가진 컴포넌트 목록
⑤ 데이터 / 페르소나 분기화면이 의존하는 입력값, 게스트 / 시민권 / 기업 페르소나별 차이
⑥ 상태 매트릭스Empty · Loading · Error · Success 4가지 상태에서의 UI 처리
⑦ CTA / 다음 화면Primary CTA, Secondary CTA, 각각의 이동 라우트

1.2 표기 규칙

  • KR한국 시민권 페르소나에서만 노출되거나 다르게 동작하는 항목.
  • FR외국인(FOREIGNER) 페르소나 기준 항목.
  • ENT기업 코드 검증 완료된 페르소나에서 추가/변경되는 항목.
  • GUEST로그인 전 게스트 상태에서 적용되는 항목.
  • SIGNED로그인 후 적용되는 항목.
  • MUST 데모 합격 기준 / SHOULD 권장 / COULD 시간 허용 시 / OUT 데모 범위 외.
