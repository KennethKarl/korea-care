01문서 개요

1.1 본 문서의 위치

본 문서는 내부 정책서이다. 사용자 노출 문서인 Privacy Notice, Terms of Service, 동의 화면 카피는 본 문서를 근거로 별도 작성되며, 외부 변호사 사인오프 후 게시된다. 본 문서가 다른 .pm/ 문서와 충돌할 경우, 본 문서를 우선한다.

1.2 적용 범위

  • 기간: 2026-05-15 데모 ~ 일반 출시 전(결제 미연동 단계).
  • 제품: SafeDoc Global Service 반응형 웹 (Next.js 프로덕션 빌드).
  • 법적 디스클레이머: 본 문서는 법률 자문이 아니다. 외부 변호사 사인오프 전에는 외부 공개 금지.

동의 위치 (가입 vs 예약 단계) 및 민감정보(진료과)와 고유식별정보(여권)의 별도 체크박스 분리 여부는 검토 대기. 동의 철회는 계정 설정 또는 privacy@safedoc.io로 요청 가능, 처리 SLA 10영업일.

04이용약관 (ToS 내부 사양)

4.1 서비스 정의

SafeDoc Global Service는 한국 병원의 선택 의료(elective care) 예약을 코디네이션하는 플랫폼이다. 의료기관이 아니며, 의료 행위를 제공하지 않는다. 의료 행위는 사용자가 선택한 한국 병원이 한국법에 따라 직접 수행한다.

4.2 회원 자격

  • 만 18세 이상.
  • 미국 거주(자가 신고).
  • 13세 미만 가입 차단 — 약관 명시 + 가입 시 연령 확인. 검토 대기 — COPPA 적용 범위 평가.

4.3 사용자 의무

  • 정확한 정보 제공 (성명·여권 번호·시민권 일치).
  • 여권 정보 변경 시 즉시 갱신 또는 CS 문의.
  • 병원 방문 시 시민권 증빙 제출 (KR_RESIDENT 티어 적용자 한정).

4.4 책임 한계

  • 의료 행위 자체에 대한 책임은 한국 병원에 있다.
  • SafeDoc는 예약 접수 코디네이션, 가격 정보 제공, 부가서비스 연결 책임만 진다.
  • SafeDoc는 병원의 진료 결과 · 합병증 · 의료 사고에 대해 직접 책임지지 않는다.
  • 본 면책은 미국 텍사스주법 및 한국법이 허용하는 최대 범위 내에서 적용된다.

4.5 금지 행위

  • 허위 정보 제공 (시민권 도용, 여권 번호 위조 등)
  • 타인 명의로 가입 또는 예약
  • 자동화 스크래핑, API 무단 사용
  • 플랫폼 보안 우회 시도

4.6 계정 해지

  • 사용자 요청 시 30일 이내 데이터 익명화/삭제.
  • 정당한 사유(허위 정보, 사기 의심) 발견 시 SafeDoc의 즉시 정지 권한.

4.7 분쟁 해결

  • 1차 협상 (사용자 ↔ CS, 30일 이내).
  • 미해결 시 중재: JAMS 중재 (Delaware) 검토 대기 — 데모 단계는 형식적 게시만, 실제 분쟁 발생 전 변호사 사인오프 필수.
  • 한국 본사 측 데이터 처리 분쟁은 한국법 적용.

4.8 준거법

  • Texas LLC 운영 측: 텍사스주법. 검토 대기
  • 한국 데이터 처리: 한국법.
  • 이중 적용 시 사용자에게 더 유리한 보호를 우선한다.

05가격 정책

5.1 가격 결정 트리

display_price(service, user) =
  base_price[tier]                        ← (1) 시민권 기반 티어 결정
    × (1 - enterprise_discount)           ← (2) 기업 verified 시 적용
    × (1 - coupon_discount)               ← (3) 쿠폰 코드 적용
    (단, 쿠폰과 기업 할인이 비-스택이면 사용자 라디오 선택)

5.2 티어 결정 규칙

티어적용 조건예시
FOREIGNER시민권 ≠ KRUS 시민, 비-KR 거주자
KR_RESIDENT시민권 = KR (자가 신고)한인 (한국 국적, 이중국적 포함)
+ ENTERPRISE위 둘 중 하나 + 기업 코드 verifiedACME-2026 인증 후
PENDING기업 코드 입력 후 admin 검토 대기verified 전까지는 기본 시민권 티어 적용

5.3 충돌 해소 (Coupon ↔ Enterprise)

  • 비-스택 쿠폰과 기업 할인이 동시 적용 가능한 상태 시 사용자에게 라디오로 양자택일 강제.
  • 기본 선택은 강한 할인이 자동 미리 선택되나, 사용자가 명시적으로 다른 쪽으로 변경 가능.
  • 선택 결과는 state.conflictChoice로 보존, 새로고침 후에도 유지.

5.4 표시 통화

  • 데모 단계: USD 단일.
  • Phase 2: KRW 옵션 추가 (Stripe 도입과 함께).

5.5 부가가치세 · 표시 의무

  • "예상 총액"이 최종 청구액이 아닐 수 있음을 명시 (한국 측 환율, 추가 시술, 부가서비스 정산 변동 가능).
  • 환율 변동 면책 문구 명시.
  • 한국 의료 부가세 표시 (포함/별도) 검토 대기

5.6 테스트 코드 (데모 한정)

  • SAFEDOC10 — 10% 할인 (스택 가능).
  • NEW200 — $200 할인 (비-스택, enterprise와 충돌).
  • 출시 전 코드 제거 + 빌드 가드 확인.

10보안 정책

10.1 전송 구간

  • TLS 1.3 강제 (TLS 1.2 fallback 허용, 1.1 이하 차단).
  • HSTS: max-age=31536000 (1년) + includeSubDomains + preload 검토 대기
  • HTTPS-only. HTTP 요청 시 즉시 301 redirect.

10.2 저장 구간

  • 여권 번호 · 여행 일정 등 PII는 KMS-managed key로 AES-256 암호화 (envelope encryption).
  • DB 자체는 at-rest encryption 기본 활성화 (AWS RDS / Vercel KV 동일).
  • 백업 데이터도 동일 암호화 정책 적용. 백업 보유 90일.

10.3 인증

  • Google · Apple OAuth만 (비밀번호 미보관).
  • 세션 쿠키: HttpOnly + Secure + SameSite=Lax.
  • 토큰 회전 정책: refresh token rotation, 30일 inactivity 시 강제 재로그인.
  • Phase 2 — Kakao/Naver 추가 검토.

10.4 로그 · 모니터링

  • PII 마스킹 의무. raw 여권 번호·이메일·전화는 로그에 절대 흘러가지 않게 allowlist 적용.
  • Sentry / CloudWatch / Vercel logs 모두 PII scrubber 통과 후 인입.
  • 접근 로그 1년 보관. 보안 사고 발견 시 즉시 추출 가능.

10.5 권한 관리

  • 어드민 인터페이스 접근: MFA 필수.
  • 최소권한 원칙. role-based access control (RBAC).
  • 어드민 액션 (기업 verified 처리, 사용자 데이터 조회 등): audit log 강제 + 사후 검토.
  • 개발자 production DB 직접 접근 금지 (모든 쿼리는 어드민 인터페이스 경유).

10.6 사고 대응

  1. 발견 즉시: 격리, 영향 평가 (72시간 이내).
  2. Texas HB 300: 영향 받은 개인에게 60일 이내 통보, 250명+ 영향 시 텍사스 AG에 30일 이내 통보. 검토 대기 — 외부 변호사 사인오프 후 확정.
  3. CCPA: 캘리포니아 거주자 영향 시 별도 통보 의무.
  4. PIPA: 한국 측 데이터 영향 시 한국 본사와 공동 대응.
  5. 사후 RCA + 재발 방지 plan 작성.
