# SafeDoc Global 웹 — 콘텐츠 관리(Admin) 가이드

> **주소**: https://kennethkarl.github.io/korea-care/admin/
> 사이트(https://kennethkarl.github.io/korea-care/)의 **리스트·콘텐츠를 코드 수정 없이** 직접 편집·발행하는 화면입니다.

---

## 0. 3줄 요약

1. `/admin` 에서 콘텐츠를 **편집 → 저장**하면 *내 브라우저*에서 미리보기됩니다.
2. **발행**(GitHub 토큰 필요)을 눌러야 **실제 사이트의 모든 방문자**에게 반영됩니다(~1분).
3. 발행 전까지는 아무리 저장해도 실제 사이트는 안 바뀝니다. **꼭 "발행"까지** 하세요.

---

## 1. 동작 원리 (왜 이렇게 나뉘나)

이 사이트는 서버·DB 없이 GitHub Pages로 돌아가는 **정적 사이트**입니다. 그래서 저장 단계가 2개입니다.

| 단계 | 버튼 | 저장 위치 | 누가 봄 | 반영 시점 |
|------|------|-----------|---------|-----------|
| 편집 미리보기 | **저장** | 내 브라우저(localStorage) | 나만 | 즉시 |
| 실제 발행 | **발행** | 레포 `content.json` | 전 방문자 | ~1분 후 |

- 공개 페이지는 접속 시 `content.json`(발행본)을 읽어 표시합니다. 발행본이 없으면 기본값(시드)으로 표시됩니다 → 그래서 아무것도 안 해도 사이트는 정상입니다.

---

## 2. 발행용 GitHub 토큰 만들기 (최초 1회)

발행하려면 GitHub 토큰이 필요합니다. `KennethKarl` 계정으로 로그인한 상태에서:

1. **https://github.com/settings/personal-access-tokens/new** 접속
   (경로: GitHub 프로필 → Settings → Developer settings → Personal access tokens → **Fine-grained tokens** → Generate new token)
2. **Token name**: 예) `korea-care admin`
3. **Expiration**: 원하는 만료일 (예: 90 days)
4. **Resource owner**: `KennethKarl`
5. **Repository access**: **Only select repositories** → **`KennethKarl/korea-care`** 선택
6. **Permissions → Repository permissions → Contents → `Read and write`** 로 변경 (나머지 그대로)
7. **Generate token** → 나오는 **`github_pat_…`** 문자열 **복사** (⚠️ 이 화면에서만 한 번 보입니다)
8. `/admin` 상단 **발행** 칸에 붙여넣기

> 토큰은 **내 브라우저 세션에만** 저장되고 코드·사이트엔 포함되지 않습니다. 다음 접속 시 재입력 불필요. 만료되면 같은 방법으로 새로 발급해 교체하세요.

## 4. 콘텐츠 편집하기

1. 왼쪽에서 **컬렉션 탭** 선택 (예: `시술/예약`)
2. 항목 행의 버튼:
   - **편집** — 상세 폼 열기/닫기
   - **복제** — 해당 항목을 복사해 아래에 추가 (새 항목 만들 때 편리)
   - **삭제** — 항목 제거
   - **↑ / ↓** — 노출 순서 변경
   - **+ 추가** — 빈 항목 추가
   - **시드 복원** — 그 컬렉션을 기본값으로 되돌림(로컬 편집분 사라짐)
3. **편집** 폼에서 값 수정 후 반드시 **저장** 클릭

### 필드 입력 규칙
- **EN / KO 두 칸**: 대부분 텍스트는 영문·한글 두 칸이 함께 나옵니다. **둘 다** 채우세요(한글 화면/영문 화면에 각각 노출).
- **이미지**: 이미지 주소(URL)를 입력하면 아래 미리보기가 뜹니다. (파일 업로드가 아니라 URL 방식)
- **리스트 항목**(예: 포함 내역, 단계): `+ 추가`로 줄 추가, `✕`로 삭제.
- **숫자**(예: 평점·가격): 숫자만.
- **체크박스**(예: 영어지원 여부): 켜기/끄기.

> 💡 **저장 = 미리보기**입니다. 저장 후 새 탭에서 실제 페이지(예: `/treatments`)를 새로고침하면 **내 브라우저에서는** 바뀐 걸 볼 수 있습니다. 다른 사람은 아직 못 봅니다.

## 5. 발행하기 (실제 반영)

1. 상단 **발행** 칸에 GitHub 토큰이 들어있는지 확인
2. **발행** 버튼 클릭 → 확인창 **확인**
3. "발행 완료 — GitHub Pages 전파 후 ~1분 뒤 반영됩니다" 메시지 + **커밋 보기** 링크 표시
4. **약 1분 뒤** 실제 사이트에 전 방문자 대상 반영 (브라우저 새로고침으로 확인)

### 토큰 없이 발행 (대안)
- **JSON 다운로드** 버튼으로 `content.json` 파일을 받아, `KennethKarl/korea-care` 레포 **루트**에 직접 커밋해도 동일하게 반영됩니다.

### 기타 버튼
- **로컬 편집 비우기**: 내 브라우저의 미편집분을 전부 지우고 *현재 발행본(또는 기본값)* 상태로 되돌립니다. (실제 사이트엔 영향 없음)

## 6. 자주 묻는 것 / 문제 해결

| 증상 | 원인 / 해결 |
|------|-------------|
| "GitHub 토큰을 입력하세요." | 발행 칸이 비어 있음 → §2 로 토큰 발급 후 입력 |
| "토큰이 유효하지 않음" (401) | 토큰 만료/오타 → 새로 발급 |
| "레포/경로 또는 토큰 권한 확인" (404) | 토큰 권한에 **Contents: Read and write** 누락, 또는 레포 선택 안 함 → §2 5·6번 재확인 |
| 발행했는데 사이트가 안 바뀜 | ① 아직 1분 안 지남 ② 브라우저 캐시 → 강력 새로고침(Cmd/Ctrl+Shift+R) |
| 저장했는데 다른 사람이 못 봄 | **발행**을 안 함 → 발행 필수 |
| 실수로 다 지움 | 해당 컬렉션 **시드 복원**, 또는 발행 전이면 **로컬 편집 비우기** |

## 7. 한계 / 주의 (개발자 영역)

- **새로 추가한 항목의 상세 페이지**: 목록/카드에는 바로 뜨지만, 새 시술·병원의 개별 **상세페이지(`/treatments/새ID` 등)** 는 정적 생성 대상이라 **재배포가 있어야** 만들어집니다. *기존 항목 내용 수정*은 재배포 없이 발행만으로 반영됩니다.
- **코드 재배포 시 발행본 보존**: 개발자가 사이트 코드를 재배포할 때 발행본 `content.json`이 덮이지 않도록 보존해야 합니다(누락 시 기본값으로 환원). 재배포 후에는 한 번 더 발행하면 안전합니다.
- 현재 편집 대상은 **핵심 CMS 리스트 12종**(Phase 1)입니다. 상단 배너 문구·네비게이션·푸터·약관 등은 아직 편집 대상이 아닙니다(후속 단계).
