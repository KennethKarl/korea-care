콘텐츠 관리(Admin) 가이드

🔗 kennethkarl.github.io/korea-care/admin
3줄 요약
  1. /admin에서 콘텐츠를 편집 → 저장하면 내 브라우저에서만 미리보기됩니다.
  2. 발행(GitHub 토큰 필요)을 눌러야 실제 사이트의 모든 방문자에게 반영됩니다(~1분).
  3. 저장만 하면 실제 사이트는 안 바뀝니다 — 꼭 “발행”까지 하세요.

1동작 원리 — 왜 저장/발행 2단계인가

이 사이트는 서버·DB 없이 GitHub Pages로 도는 정적 사이트라 저장 단계가 둘로 나뉩니다.

단계버튼저장 위치누가 봄반영
편집 미리보기저장내 브라우저(localStorage)나만즉시
실제 발행발행레포 content.json전 방문자~1분 후

공개 페이지는 접속 시 content.json(발행본)을 읽어 표시하고, 없으면 기본값(시드)으로 표시합니다 → 아무것도 안 해도 사이트는 정상입니다.

2발행용 GitHub 토큰 만들기 (최초 1회)

KennethKarl 계정으로 GitHub에 로그인한 상태에서:

  1. github.com/settings/personal-access-tokens/new 접속
    경로: 프로필 → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token
  2. Token name: 예) korea-care admin
  3. Expiration: 원하는 만료일 (예: 90 days)
  4. Resource owner: KennethKarl
  5. Repository access → Only select repositories → KennethKarl/korea-care 선택
  6. Permissions → Repository permissions → Contents → Read and write 로 변경 (나머지 그대로)
  7. Generate token → 나오는 github_pat_… 문자열 복사
  8. /admin 상단 발행 칸에 붙여넣기
⚠️ 토큰 문자열은 생성 직후 한 번만 보입니다. 창을 닫기 전에 복사하세요(놓치면 새로 발급).
🔒 붙여넣은 토큰은 내 브라우저 세션에만 저장되고 코드·사이트엔 포함되지 않습니다. 다음 접속 시 재입력 불필요.

4콘텐츠 편집하기

  1. 왼쪽 컬렉션 탭 선택 (예: 시술/예약)
  2. 항목 행 버튼 — 편집(폼 열기) · 복제(복사해 아래 추가) · 삭제 · ↑ ↓(순서) / 상단 + 추가 · 시드 복원(기본값 되돌리기)
  3. 편집 폼에서 수정 후 반드시 저장 클릭

필드 입력 규칙

  • EN / KO 두 칸: 텍스트는 영문·한글 두 칸이 함께 나옵니다 — 둘 다 채우세요(각 언어 화면에 노출).
  • 이미지: 이미지 주소(URL) 입력 → 아래 미리보기 표시(파일 업로드 아님).
  • 리스트(포함 내역·단계 등): + 추가로 줄 추가, ✕로 삭제.
  • 숫자(평점·가격 등)는 숫자만, 체크박스(영어지원 등)는 켜기/끄기.
💡 저장 = 미리보기. 저장 후 새 탭에서 실제 페이지(예 /treatments)를 새로고침하면 내 브라우저에서는 바뀐 게 보입니다. 다른 사람은 아직 못 봅니다.

5발행하기 (실제 반영)

  1. 상단 발행 칸에 토큰이 들어있는지 확인
  2. 발행 클릭 → 확인창 확인
  3. “발행 완료 …” 메시지 + 커밋 보기 링크 표시
  4. 약 1분 뒤 전 방문자 대상 반영 (새로고침으로 확인)

토큰 없이 발행 (대안)

JSON 다운로드로 content.json을 받아 KennethKarl/korea-care 레포 루트에 직접 커밋해도 동일하게 반영됩니다.

로컬 편집 비우기: 내 브라우저의 미편집분을 지우고 현재 발행본(또는 기본값)으로 되돌립니다(실제 사이트엔 영향 없음).

6문제 해결

증상원인 / 해결
“GitHub 토큰을 입력하세요.”발행 칸이 빔 → §2로 토큰 발급 후 입력

발행했는데 안 바뀜① 아직 1분 안 지남 ② 캐시 → 강력 새로고침 Cmd/Ctrl+Shift+R
저장했는데 다른 사람이 못 봄발행을 안 함 → 발행 필수
실수로 다 지움컬렉션 시드 복원, 또는 발행 전이면 로컬 편집 비우기

7한계 / 주의 (개발자 영역)

  • 새로 추가한 항목의 상세 페이지: 목록·카드엔 바로 뜨지만, 새 시술·병원의 개별 상세페이지(/treatments/새ID 등)는 정적 생성 대상이라 재배포가 있어야 만들어집니다. 기존 항목 수정은 발행만으로 반영됩니다.
  • 코드 재배포 시 발행본 보존: 사이트 코드 재배포 때 content.json이 덮이지 않도록 보존 필요(누락 시 기본값 환원). 재배포 후 한 번 더 발행하면 안전.
  • 현재 편집 대상은 핵심 CMS 리스트 12종(Phase 1). 상단 배너·네비게이션·푸터·약관은 아직 대상 아님(후속 단계).

8요약 체크리스트

  • GitHub 토큰 발급 (§2, 최초 1회)
  • /admin 접속 → 컬렉션 선택 → 편집 → 저장
  • 미리보기 확인 (내 브라우저에서 실제 페이지 새로고침)
  • 발행 클릭 → ~1분 후 실제 사이트 확인
